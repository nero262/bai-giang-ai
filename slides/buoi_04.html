4 TỪ KHOÁ CẦN NHỚ

Bộ từ khoá buổi trước

LLM
Mô hình ngôn ngữ lớn — học từ rất nhiều văn bản để đoán token tiếp theo, tạo câu trả lời. Rất trôi chảy nhưng có thể sai.
Token
Đơn vị nhỏ mô hình dùng để đọc/tạo văn bản: một từ, phần của từ hoặc ký tự. Câu quá dài → nhiều token.
RAG
Tìm thông tin từ nguồn bên ngoài trước, rồi mới trả lời có căn cứ. VD: NotebookLM đọc file thầy cô gửi.
Vibe Coding
Mô tả ý tưởng bằng lời → AI viết code → bạn chạy thử, ghi lỗi, yêu cầu sửa qua nhiều vòng.
🔎

Kiểm chứng: dù dùng AI cách nào, bạn vẫn phải chạy thử, đọc lỗi và đối chiếu nguồn — AI không "biết chắc sự thật".

GÓI NĂNG LỰC CHUYÊN BIỆT

Skill là gì?

🧩

Skill là một gói hướng dẫn chuyên biệt giúp AI làm tốt một loại nhiệm vụ. Skill thường chứa: mô tả nhiệm vụ, quy trình, quy tắc đầu ra, ví dụ, template và các bước kiểm tra.

🎮 HTML Game Builder
Hướng dẫn AI tạo game web có canvas, vòng lặp game, xử lý bàn phím & checklist kiểm thử.
🎨 UI Designer
Hướng dẫn AI tạo giao diện đẹp, bố cục rõ, nút dễ dùng, màu nhất quán.
📺 HTML Presentation
Hướng dẫn AI tạo slide HTML nhiều màn hình, có chuyển cảnh & điều khiển bằng phím.
3 MỨC ĐỘ LÀM VIỆC VỚI AI

Prompt → Skill → Agent

Prompt = câu nhờ việc · Skill = bản hướng dẫn nghề · Agent = người làm qua nhiều bước (bạn vẫn kiểm soát mục tiêu & quyền).

KHÔNG CẦN XUẤT FILE .MP4

Slide HTML "dạng video" là gì?

📺

Là một trang HTML gồm nhiều màn hình slide, có hiệu ứng chuyển cảnh & tự động chuyển sau vài giây — tạo cảm giác như video giới thiệu. Cần video thật thì quay màn hình lúc slide tự chạy.

Transition
Hiệu ứng chuyển cảnh giữa slide: mờ dần, trượt sang.
Auto-play
Tự chuyển slide sau ~8 giây, có nút bật/tắt.
Animation

BẠN CẦN NHỚ

8 ý chính của buổi 4

1

Skill = bộ hướng dẫn có cấu trúc, dùng lại nhiều lần, ổn định hơn prompt.

2

Prompt → Skill → Agent là 3 mức; bạn vẫn kiểm soát mục tiêu & quyền.

3

Skill thường viết bằng Markdown (.md) — dễ đọc với cả người & AI.

4

"Code chạy được" chưa phải sản phẩm hoàn chỉnh.

5

Đóng gói = ý tưởng → giao diện → code → kiểm thử → chia sẻ → cải tiến.

6

IDE gộp viết mã, chạy thử, đọc lỗi, debug; HTML/CSS/JS = cấu trúc/đẹp/logic.

7

Mô tả lỗi cụ thể theo mẫu X–Y–Z để AI sửa đúng.

8

Luôn tự kiểm thử — đừng tin "AI nói đã sửa xong".

BẤM ĐỂ LẬT THẺ

Flashcard ôn tập nhanh

Khái niệmSkill là gì?
Bộ hướng dẫn có cấu trúc để AI làm tốt một nhóm nhiệm vụ lặp lại — quy tắc + checklist.
Khái niệmAgent là gì?
Trợ lý AI nhận mục tiêu, lập kế hoạch, dùng công cụ, làm nhiều bước & tự kiểm tra (bạn kiểm soát quyền).
Định dạngMarkdown?
Định dạng văn bản thuần (.md) dùng ký hiệu đơn giản (#, -, ```) — dễ đọc với cả người & AI.
Khái niệmĐóng gói sản phẩm?
Biến ý tưởng/đoạn code thành sản phẩm có thể dùng, kiểm tra, chia sẻ & bảo trì.
Công cụIDE là gì?
Môi trường phát triển tích hợp: viết mã, chạy thử, đọc lỗi, debug trong một nơi. VD: VS Code, Cursor.
Ngôn ngữHTML · CSS · JS?
HTML = cấu trúc · CSS = làm đẹp giao diện · JavaScript = tương tác & logic.
