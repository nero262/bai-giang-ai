HAI CÂU HỎI QUEN THUỘC

"Web chạy trên máy em, sao bạn mở link lại trắng trơn?"

Và: em với bạn ở hai đầu thành phố, sao lại cùng xem một video, cùng chơi một trận game online? Câu trả lời cho cả hai chỉ nằm ở đúng một chữ: SERVER.

localhost ≠ internet→ Client & Server
BỨC TRANH LỚN

Internet = hàng tỉ máy tính nối với nhau

Internet không phải "đám mây" mơ hồ. Nó là một mạng lưới máy tính thật khắp thế giới, nối bằng dây cáp (kể cả cáp ngầm dưới biển!) và sóng, để trao đổi dữ liệu cho nhau.

ẨN DỤ GAME

Giống một server Minecraft/Roblox khổng lồ mà cả thế giới cùng vào: máy của mỗi người kết nối tới các "máy chủ" chung để cùng thấy một thế giới, cùng chơi với nhau.

GHI NHỚ

Không có gì "biến mất trong không khí". Mọi thứ em xem đều nằm trên một máy tính thật ở đâu đó, và được gửi tới em.

HAI VAI TRÒ CỐT LÕI

Client & Server — ai hỏi, ai trả lời?

📱
Client
= Máy của em (người chơi)
Điện thoại/laptop của em, chạy app hoặc trình duyệt. Nó đi hỏi và hiển thị kết quả.
🗄️
Server
= Máy chủ luôn bật
Một máy tính mạnh, bật 24/7, giữ dữ liệu (video, tin nhắn, tài khoản) và trả lời khi được hỏi.
🌐
Internet
= Con đường nối 2 bên
Đường truyền để client và server gửi dữ liệu qua lại cho nhau.
ẨN DỤ

Client như người chơi bấm nút; Server như máy chủ game giữ cả thế giới và trả lời mọi người chơi cùng lúc.

MỘT LẦN "HỎI — ĐÁP"

Request → Response: chuyện xảy ra khi em bấm

  1. Em mở app / gõ địa chỉ → Client gửi một Request ("cho tôi xin video này").
  2. Request đi qua Internet tới đúng Server giữ dữ liệu đó.
  3. Server xử lý, rồi gửi lại một Response (chính video/dữ liệu đó).
  4. Client nhận Response và hiển thị lên màn hình cho em xem.
ẨN DỤ QUÁN ĂN

Em (client) gọi món (request) → bếp (server) làm xong bưng ra (response). Cả tỉ lần "gọi món – bưng ra" mỗi giây chính là internet đang chạy.

TRẢ LỜI CÂU HỎI ĐẦU BUỔI

Server = "điểm hẹn" nối mọi người lại

💬

🎮

Chơi chung (Game online)

Mọi người chơi cùng vào 1 server → thấy cùng một trận đấu, cùng lúc.

CHÌA KHOÁ

Mọi người không nối thẳng máy này với máy kia. Tất cả gặp nhau ở server chung — đó là cách các ứng dụng kết nối cả thế giới.

LÀM SAO TÌM ĐÚNG SERVER?

Domain · IP · DNS — địa chỉ của server

🏷️
Domain
= Tên dễ nhớ
Tên miền như tiktok.com — dễ đọc, dễ nhớ (giống tên quán).
🔢
IP
= Số nhà thật
Địa chỉ số của server (vd 142.250.x.x) — máy tính dùng cái này để tìm nhau.
📖
DNS
= Danh bạ
Tra tên miền → số nhà IP. Em gõ tên, DNS tìm ra đúng server.
GỘP LẠI

Gõ tiktok.com → DNS tra ra IP server → client gửi request tới đúng "số nhà" đó.

CÁCH XIN DỮ LIỆU TỪ SERVER

API — "quầy nhận đơn" của server

Server không cho em vào lục đồ lung tung. Nó mở sẵn một API — cái "quầy" có ô nhận yêu cầu và trả về dữ liệu gọn gàng (thường ở dạng JSON).

ẨN DỤ QUẦY ORDER

API giống quầy order ở quán: em đưa đúng yêu cầu qua ô cửa (địa chỉ API) → server đưa lại đúng phần dữ liệu. Em không cần biết bếp nấu thế nào.

TRONG CODE

Trình duyệt gọi API bằng lệnh fetch("địa-chỉ-api") → nhận về dữ liệu → hiển thị. (AI sẽ viết phần này cho em.)
