HAI VAI TRÒ CỐT LÕI

Client & Server — ai hỏi, ai trả lời?

📱
Client
= Máy của em (người chơi)
Điện thoại/laptop của em, chạy app hoặc trình duyệt. Nó đi hỏi và hiển thị kết quả.
🗄️
Server
= Máy chủ luôn bật
Một máy tính mạnh, bật 24/7, giữ dữ liệu (video, tin nhắn, tài khoản) và trả lời khi được hỏi.
🌐
Internet
= Con đường nối 2 bên
Mạng lưới máy tính thật khắp thế giới (cả cáp ngầm dưới biển!) để client và server gửi dữ liệu qua lại.
ẨN DỤ GAME

Client như người chơi bấm nút; Server như máy chủ game (Minecraft/Roblox) giữ cả thế giới và trả lời mọi người chơi cùng lúc.

MỘT LẦN "HỎI — ĐÁP"

Request → Response: chuyện xảy ra khi em bấm

  1. Em mở app / gõ địa chỉ → Client gửi một Request ("cho tôi xin video này").
  2. Request đi qua Internet tới đúng Server giữ dữ liệu đó.
  3. Server xử lý, rồi gửi lại một Response (chính video/dữ liệu đó).
  4. Client nhận Response và hiển thị lên màn hình cho em xem.
ẨN DỤ QUÁN ĂN

Em (client) gọi món (request) → bếp (server) làm xong bưng ra (response). Cả tỉ lần "gọi món – bưng ra" mỗi giây chính là internet đang chạy.

TRẢ LỜI CÂU HỎI ĐẦU BUỔI

Server = "điểm hẹn" nối mọi người lại

💬

Nhắn tin (Zalo)

Tin của em đi máy em → server Zalo → máy bạn. Server là trạm trung chuyển ở giữa.

🎬

Cùng xem (TikTok)

Video nằm trên 1 server chung; triệu người cùng "gọi" nó → ai cũng xem được cùng một thứ.

🎮

Chơi chung (Game online)

Mọi người chơi cùng vào 1 server → thấy cùng một trận đấu, cùng lúc.

CHÌA KHOÁ

Mọi người không nối thẳng máy này với máy kia. Tất cả gặp nhau ở server chung — đó là cách các ứng dụng kết nối cả thế giới.

THỰC HÀNH — TỪ REPO → LINK SỐNG

Deploy repo GitHub lên Vercel trong 5 bước

  1. Vào vercel.com → Sign Up → chọn Continue with GitHub (nối tài khoản buổi 14).
  2. Bấm Add New… → Project → chọn repo (vd game-ran-cua-em) → Import.
  3. Để nguyên cấu hình mặc định → bấm Deploy → chờ Vercel build ~1 phút.
  4. Xong! Nhận link https://<tên-app>.vercel.app — có sẵn HTTPS, sống 24/7.
  5. Mở link trên điện thoại → app của em chạy thật, dù laptop đã tắt!
CƠ BẢN — CHỈ DÙNG CHUỘT

Toàn bộ 5 bước làm trên trang vercel.com, không cần gõ lệnh. Kẹt bước nào → hỏi AI trợ lý deploy (slide sau).

ĐƯỜNG ỐNG TỰ ĐỘNG

Sửa code → git push → Vercel tự đăng bản mới

  1. Em sửa code trên máy như bình thường (vd đổi màu, thêm màn game).
  2. git push lên repo GitHub (đúng thao tác buổi 14).
  3. Vercel tự phát hiện có bản mới → build lại → cập nhật link sống.
  4. Vài chục giây sau, link .vercel.app đã là phiên bản mới nhất — không phải deploy tay lần nữa.
ẨN DỤ

Như một đường ống tự động nối từ code tới web sống: em chỉ cần push, phần "xuất bản" Vercel lo hết. Git (buổi 14) + Vercel (hôm nay) ghép thành 1 quy trình hoàn chỉnh.

GIẤU BÍ MẬT ĐÚNG CÁCH

Environment Variable — cất "chìa khoá" API an toàn

Nhiều API cần 1 chìa khoá bí mật (API key). Repo của em công khai (buổi 14) — ai cũng đọc được code, nên tuyệt đối không dán key thẳng vào code.

ĐỪNG BAO GIỜ

Dán API key vào file code rồi push lên GitHub công khai = đưa chìa khoá nhà cho cả thế giới. Bot sẽ quét thấy và lạm dụng.

NÂNG CAO — CÁCH ĐÚNG

Trên Vercel: Settings → Environment Variables → thêm tên + giá trị chìa khoá. Vercel giữ kín, code chỉ gọi tên biến. Nếu API em dùng không cần key (như slide thực hành) thì bỏ qua bước này.

CÁCH XIN DỮ LIỆU TỪ SERVER

API — "quầy nhận đơn" của server

Server không cho em vào lục lung tung. Nó mở sẵn một API — cái "quầy" có ô nhận yêu cầu và trả về dữ liệu gọn gàng (thường ở dạng JSON). Trình duyệt gọi API bằng lệnh fetch("địa-chỉ-api").

🐶

Ảnh vui (dễ nhất)

Bấm nút → hiện 1 ảnh chó ngẫu nhiên từ server.

dog.ceo/api/breeds/image/random

❓

Quiz học tập

Bấm nút → hiện 1 câu hỏi trắc nghiệm ngẫu nhiên từ server.

opentdb.com/api.php?amount=1&type=multiple

Gọi thẳng từ trình duyệt là có dữ liệu sống. (opentdb trả câu hỏi tiếng Anh — chưa quen thì chọn ảnh chó cho nhẹ.)
