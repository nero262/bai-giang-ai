4 nhóm AI — OpenCode đứng ở đâu? 🗺️

Cùng nhìn lại bản đồ công cụ từ Buổi 9 — lần này tập trung vào điểm khác biệt của CLI Agent:

Nhóm 1
Chat-first AI
  • ChatGPT, Claude Web
  • Trả ra văn bản/code rời rạc
  • Người dùng tự copy-paste
Nhóm 2
Platform AI
  • Bolt.new, v0.dev
  • Prototype nhanh
  • Bị giới hạn trong Cloud sandbox
Nhóm 3
IDE AI
  • Cursor, Copilot
  • Gợi ý code trong editor
  • Người dùng phải bấm Accept từng đoạn
⭐
Nhóm 4 — Đỉnh tháp
CLI Agent
  • OpenCode, Aider
  • Toàn quyền hệ thống
  • Tự thực thi lệnh · Đọc log · Tự sửa
⚡

CLI Agent là nhóm DUY NHẤT có thể chạy lệnh hệ thống, cài thư viện, đọc log lỗi và tự sửa — tất cả không cần bạn bấm nút.

BẢN CHẤT CLI AGENT

Vòng lặp tự trị — AI làm thay bạn cả chuỗi 🔄

Trong khi IDE AI chờ bạn bấm Accept, CLI Agent vận hành theo vòng lặp khép kín:

Nhận yêu cầu
→
Duyệt file
→
Viết code
→
Chạy lệnh
→
Đọc log lỗi
→
Tự sửa 🔁
⚠️

Với IDE AI: Bạn phải bấm Run → đọc lỗi → copy sang chat → đọc gợi ý → bấm Accept → lặp lại. Mỗi vòng mất 2–5 phút.

✅

Với CLI Agent: Bạn gõ yêu cầu → Agent tự chạy cả vòng lặp đến khi hết lỗi. Bạn chỉ việc đọc kết quả cuối.

Terminal & Cài đặt OpenCode

Làm quen bàn phím của Power User — rồi nạp AI vào trong đó

NỀN TẢNG

Terminal — Bàn phím điều khiển máy tính 💻

Terminal (Giao diện dòng lệnh)
Cửa sổ bằng chữ để ra lệnh cho máy tính · Trên Windows: CMD, PowerShell
Terminal Emulator
Phần mềm hiện đại: WezTerm, Alacritty · Phông đẹp hơn, nhiều tab, nhanh hơn nhờ GPU
Shell (Bộ thông dịch)
Chương trình xử lý lệnh bạn gõ · PowerShell (Windows) · Bash/Zsh (Linux/Mac)
💡

Hãy nghĩ Terminal như điều khiển từ xa của máy tính — thay vì bấm vào biểu tượng trên màn hình, bạn gõ tên lệnh và máy tính thực hiện ngay.

⚠️

Người dùng Windows nên chạy OpenCode trên WSL (Windows Subsystem for Linux) — môi trường Linux thu nhỏ ngay trong Windows, giúp OpenCode tìm file nhanh hơn và tương thích tốt hơn.

KỸ NĂNG SỐNG SÓT

4 lệnh bạn PHẢI biết trước khi dùng OpenCode ⌨️

1

Xem danh sách tệp tin — dir (Windows) / ls (Mac/Linux)

Biết mình đang đứng ở thư mục nào, có những gì bên trong

2

Tạo thư mục mới — mkdir ten-thu-muc

Tạo thư mục chứa dự án trước khi bắt đầu build

3

Di chuyển thư mục — cd ten-thu-muc | cd .. (quay ra)

Đi vào thư mục dự án để OpenCode làm việc đúng chỗ

4

Làm sạch màn hình — cls (Windows) / clear (Mac/Linux)

Xóa các dòng cũ để dễ nhìn kết quả mới nhất

DEMO — Tạo thư mục và khởi động OpenCode ~$ mkdir quiz-app ~$ cd quiz-app ~/quiz-app$ opencode ✓ OpenCode đã khởi động · gõ @ để chọn file · Tab để chuyển mode

Ask → Plan → Build — 3 bước kiểm soát rủi ro 🎯

Nhấn [Tab] để chuyển giữa các mode. Code chỉ thay đổi khi bạn vào Build:

Bước 1
ASK
@ + Enter
  • Nhấn @ để chọn file cần xử lý
  • Mô tả yêu cầu bằng ngôn ngữ tự nhiên
  • AI hỏi lại nếu chưa rõ
  • Chưa thay đổi gì
Tab ▶
Bước 2
PLAN
[Tab]
  • Agent đề xuất kiến trúc & các bước
  • Chỉ có văn bản — chưa có code
  • Bạn duyệt và chỉnh kế hoạch
  • An toàn tuyệt đối
Tab ▶
Bước 3
BUILD
[Tab]
  • Agent viết code + chạy kiểm tra
  • Đọc log lỗi → tự sửa
  • Chạy đến khi không còn lỗi
  • Gõ /undo để hoàn tác
🎯

Mẹo quan trọng: Luôn duyệt Plan kỹ trước khi bấm Tab sang Build. 5 phút đọc Plan = tiết kiệm 30 phút sửa lỗi.

OpenCode trong hành động — ví dụ thực 🎬

ASK MODE — Yêu cầu Tạo script Python đọc file Excel và lọc các dòng có cột 'Điểm' > 8, lưu kết quả ra file mới. File gốc: /data/bảng-điểm.xlsx [Tab] → chuyển sang PLAN...

⏱️

Đây là ví dụ thực tế: mất khoảng 90 giây từ lúc gõ yêu cầu đến khi có file Excel kết quả. Thủ công có thể mất 20–30 phút.
