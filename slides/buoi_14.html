4 BƯỚC LẶP ĐI LẶP LẠI

Vòng làm việc cơ bản với Git

  1. Sửa code trong dự án như bình thường.
  2. git add — chọn những thay đổi muốn lưu vào mốc này.
  3. git commit — chốt mốc, ghi lời nhắn "vừa làm gì" (vd: "thêm màn 2").
  4. git push — đẩy mốc lên GitHub cho an toàn & chia sẻ.
TERMINAL (NÂNG CAO / ai thích dòng lệnh)git add . git commit -m "them tinh diem cho game" git push
CƠ BẢN — DỄ HƠN

Không cần gõ lệnh: dùng GitHub Desktop hoặc nút bấm ngay trên github.com — cùng ý tưởng add → commit → push nhưng bằng chuột.

THỰC HÀNH — TẠO REPO

Đưa dự án lên GitHub trong 5 bước

  1. Tạo tài khoản tại github.com (dưới 16 tuổi: đăng ký cùng phụ huynh).
  2. Bấm New repository → đặt tên (vd game-ran-cua-em) → Create.
  3. Kéo-thả các file dự án vào, hoặc bấm Add file → Upload files.
  4. Viết lời nhắn commit (vd "phiên bản đầu tiên") → Commit changes.
  5. Xong! Repo của em giờ có 1 link chia sẻ được cho cả thế giới.
MẸO

Đặt repo ở chế độ Public để khoe & để bật GitHub Pages được; Private nếu chưa muốn ai xem.

BỘ MẶT CỦA REPO

VÌ SAO CẦN

README nói rõ dự án là gì, chạy sao, ai làm + 1 ảnh/GIF sản phẩm → trang repo như một "landing page". Không có README, repo như quán không biển hiệu.

BIẾN REPO THÀNH WEB SỐNG

GitHub Pages — 1 nút bật, có link để khoe

  1. Vào repo → Settings → mục Pages.
  2. "Source": chọn nhánh main, thư mục / (root) → Save.
  3. Đợi ~1 phút, GitHub hiện link: https://<tên-em>.github.io/<tên-repo>/
  4. Mở link trên điện thoại → game của em chạy thật trên internet!
ĐÂY LÀ THỨ ĐỂ KHOE 🎉

Repo là "kho code"; link GitHub Pages là sản phẩm SỐNG — bạn bè, bố mẹ mở ngay trên điện thoại, không cần cài gì. Kẹt bước nào → hỏi AI (prompt "chỉ đường thao tác").

LƯU Ý QUAN TRỌNG

GitHub Pages CHỈ chạy được web TĨNH (HTML/CSS/JS thuần) — app "động" (có xử lý phía server, giấu chìa khoá API...) đưa lên đây sẽ không hoạt động. App động cần deploy bằng Vercel (buổi 15).

Tìm & đánh giá nhanh 1 repo

  1. Tìm: gõ từ khoá tiếng Anh vào ô Search ("snake game javascript"), lọc theo ngôn ngữ & chủ đề.

  3. Nhìn Stars & ngày cập nhật: nhiều sao + mới cập nhật = đáng tin; repo bỏ hoang 5 năm thường đã lỗi thời.

QUY TẮC AN TOÀN

Tìm file LICENSE trong repo. Không rõ → coi như không được phép. Học hỏi ý tưởng thì luôn ổn; copy nguyên code cần đúng license.

KIM CHỈ NAM KHI TÁI DÙNG

Học ý tưởng rồi tự viết lại → luôn được. Dùng lại code (đúng license) → ghi tên tác giả + link trong README. Không nhận vơ code người khác là của mình.
